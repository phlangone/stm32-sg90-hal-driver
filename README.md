# STM32 Servo Library

A small and simple STM32 HAL-based library for controlling a SG90 servo motor using a PWM channel generated by an STM32 timer.

This library is intended for educational and small embedded projects where STM32CubeMX is responsible for configuring the timer, GPIO, PWM channel, prescaler, and period. The library only provides a cleaner API to initialize a servo object, start or stop PWM, and set the servo angle.

## Features

- Uses STM32 HAL
- Uses a timer PWM channel configured by STM32CubeMX
- Provides a simple `Servo_t` object
- Starts and stops the PWM signal
- Sets the servo angle from 0 to 180 degrees
- Maps angle values linearly to timer compare values
- Keeps the application code cleaner and easier to read
- Designed for SG90 servo experiments and educational projects

## Concept

A SG90 servo is controlled by a periodic PWM signal.

In this library, the PWM period is configured in STM32CubeMX, and the library only changes the timer compare value, also called the `CCR` value.

The relationship used by the library is:

```text
angle = 0 degrees    -> SERVO_MIN_PERIOD
angle = 180 degrees  -> SERVO_MAX_PERIOD
```

The compare value is calculated linearly:

```text
compare = min_period + angle * (max_period - min_period) / 180
```

With the default values:

```c
#define SERVO_MIN_PERIOD    60U
#define SERVO_MAX_PERIOD    240U
```

the mapping becomes:

```text
0 degrees    -> CCR = 60
90 degrees   -> CCR = 150
180 degrees  -> CCR = 240
```

These values depend on the timer configuration used in STM32CubeMX.

## Requirements

- STM32 microcontroller
- STM32CubeIDE or STM32CubeMX generated project
- STM32 HAL drivers enabled
- One timer configured for PWM generation
- One PWM output pin connected to the servo signal input
- External power supply for the servo when required

## Wiring

```text
PWM pin    ---- Servo signal input
5V  supply ---- Servo VCC
GND supply ---- Servo GND
```

## STM32CubeMX configuration

### 1. Select a timer

Choose a timer that supports PWM output, for example:

- TIM1
- TIM2
- TIM3
- TIM4

The available timers and PWM pins depend on the STM32 family and package.

### 2. Enable a PWM channel

In STM32CubeMX:

1. Open the selected timer.
2. Select one timer channel.
3. Set the channel mode to **PWM Generation CHx**.
4. Configure the GPIO pin assigned to that channel.

Example:

```text
Timer: TIM2
Channel: TIM_CHANNEL_4
Mode: PWM Generation CH4
```

### 3. Configure prescaler and period

The library assumes that the timer compare values are compatible with the default servo range:

```c
SERVO_MIN_PERIOD = 60
SERVO_MAX_PERIOD = 240
```

For example, if your timer is configured so that one timer count is equivalent to 10 microseconds:

```text
CCR = 60  -> 600 us
CCR = 240 -> 2400 us
```

This can be suitable for many SG90 servo experiments.

A typical educational setup may use:

```text
Counter Period = 2000 - 1
Prescaler      = configured according to the timer clock
PWM Pulse      = 0
```

The exact prescaler depends on the STM32 clock tree and timer clock.

### 4. Set PWM polarity

A typical configuration is:

```text
PWM Mode: PWM mode 1
Pulse: 0
Output Compare Polarity: High
Fast Mode: Disable
```

### 5. Generate code

Generate the project from STM32CubeMX or STM32CubeIDE.

The generated project should contain a timer handle, for example:

```c
TIM_HandleTypeDef htim2;
```

## Installation

Copy the library files into your STM32 project:

```text
Inc/servo.h
Src/servo.c
```

Then include the header in your `main.c`:

```c
#include "servo.h"
```

Make sure the `Src` and `Inc` folders are included in your build. In STM32CubeIDE, files placed inside the project `Src` and `Inc` folders are usually detected automatically.

## API

### Servo object

```c
Servo_t servo;
```

### Initialization

```c
void Servo_Init(Servo_t *servo, TIM_HandleTypeDef *htim, uint32_t channel);
```

Initializes the servo object and links it to a timer handle and PWM channel generated by STM32CubeMX.

Example:

```c
Servo_Init(&servo, &htim2, TIM_CHANNEL_4);
```

### Start

```c
HAL_StatusTypeDef Servo_Start(Servo_t *servo);
```

Starts the PWM signal used to control the servo.

Example:

```c
Servo_Start(&servo);
```

Internally, this calls:

```c
HAL_TIM_PWM_Start(servo->htim, servo->channel);
```

### Stop

```c
HAL_StatusTypeDef Servo_Stop(Servo_t *servo);
```

Stops the PWM signal.

Example:

```c
Servo_Stop(&servo);
```

### Set angle

```c
void Servo_SetAngle(Servo_t *servo, uint8_t angle);
```

Sets the servo angle in degrees.

Valid range:

```text
0 to 180 degrees
```

Values greater than 180 are saturated to 180.

Example:

```c
Servo_SetAngle(&servo, 90);
```

## Basic example

```c
#include "main.h"
#include "servo.h"

extern TIM_HandleTypeDef htim2;

Servo_t servo;

int main(void)
{
    HAL_Init();

    SystemClock_Config();

    MX_GPIO_Init();
    MX_TIM2_Init();

    Servo_Init(&servo, &htim2, TIM_CHANNEL_4);
    Servo_Start(&servo);

    Servo_SetAngle(&servo, 0);

    while (1)
    {
        Servo_SetAngle(&servo, 0);
        HAL_Delay(500);

        Servo_SetAngle(&servo, 90);
        HAL_Delay(500);

        Servo_SetAngle(&servo, 180);
        HAL_Delay(500);
    }
}
```

## Sweep example

```c
uint8_t angle;

while (1)
{
    for (angle = 0; angle <= 180; angle += 10)
    {
        Servo_SetAngle(&servo, angle);
        HAL_Delay(200);
    }

    for (angle = 180; angle > 0; angle -= 10)
    {
        Servo_SetAngle(&servo, angle);
        HAL_Delay(200);
    }
}
```

## Notes

### Timer configuration matters

This library does not calculate pulse width in microseconds. It assumes that the timer has already been configured correctly in STM32CubeMX.

The values:

```c
SERVO_MIN_PERIOD
SERVO_MAX_PERIOD
```

are timer compare values, not necessarily microseconds.

### Adjusting the servo range

If your servo does not reach the expected mechanical range, adjust:

```c
#define SERVO_MIN_PERIOD    60U
#define SERVO_MAX_PERIOD    240U
```

according to your timer configuration and servo behavior.

### Avoid unsigned wraparound

When modifying an angle using positive and negative increments, use a signed variable:

```c
int16_t angle = 0;
```

Avoid:

```c
uint8_t angle = 0;
```

because subtracting below zero causes unsigned wraparound.

### Servo power

If the servo moves erratically, resets the board, or jitters heavily, check the power supply. Servos can draw much more current than an STM32 board pin or USB port can safely provide.
